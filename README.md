# hauler-airgap-updater

Build and deliver in-place update artifacts for fully air-gapped RKE2 clusters.
See [DESIGN.md](DESIGN.md) for the full design and rationale.

The two sides are completely disconnected. `build` (internet side) produces
**two artifacts** carried across the gap:

- `rpms_<date>.tar.zst` — rke2 RPMs in `el8/`+`el9/` dirs → the **dnf repo
  server** (nodes already point at it).
- `hauler_<date>.tar.zst` — images + helm charts → the **hauler server(s)**.

`build` also writes **`RUNBOOK.md`** — version-pinned, single-line, copy/paste
commands for loading the artifacts and applying the upgrade. The tool only
*downloads*; loading the hauler store and applying on the nodes are manual steps
the runbook spells out.

Python 3, stdlib only. Shells out to `kubectl` / `helm` / `hauler` / `tar`.

## Usage

### 1. discover (internet side, against a reachable reference cluster)
```
./airgap_update.py discover            # writes state.json (review/edit it)
```
Reads RKE2 version (`kubectl get nodes`) and helm-dep versions (`helm list`).
The reference cluster is a *proxy* — confirm the real air-gap cluster's current
version and edit `state.json` before building.

### 2. build (internet side)
```
./airgap_update.py build --dry-run                 # resolve + preview, no downloads
./airgap_update.py build --target 1.34             # interactive deps, real build
./airgap_update.py build --target 1.34 --deps-latest --no-input
```
- `--current TAG`  override the assumed current RKE2 version
- `--target MINOR` e.g. `1.34` (skip the interactive picker)
- `--deps-latest`  take latest for all helm deps
- `--dry-run`      write `airgap_hauler.yaml` + print the RPM list, then stop

On success it writes `RUNBOOK.md` (regenerate any time with
`./airgap_update.py runbook`).

### 3. follow RUNBOOK.md (manual delivery + apply)
`RUNBOOK.md` is generated for the exact path and ELs you built, with single-line
commands. In short:

- **RPMs** → on the dnf repo server, `tar -xf` (yields per-EL dirs); drop them in
  wherever your repo serves each releasever and refresh metadata your usual way.
- **Images/charts** → on **each** hauler server (internet-connected and each
  air-gapped one):
  ```
  sudo hauler store load -f hauler_<date>.tar.zst -s /opt/hauler/store
  sudo systemctl restart hauler@registry hauler@fileserver
  ```
- **Nodes** → **one minor at a time**, version-pinned (no blind `dnf update` —
  it would skip intermediate minors). Servers one-by-one (wait `Ready`/etcd),
  then agents:
  ```
  sudo dnf install -y rke2-server-<ver> rke2-common-<ver> rke2-selinux-<sel>
  sudo systemctl restart rke2-server     # agents: rke2-agent-<ver> / rke2-agent
  ```
  ⚠️ On a multi-server cluster, finish each step across a **majority of servers**
  — a partial control-plane upgrade gets reconciled back down by the lagging
  servers.

## Files
- `config.json` — static config (server paths, `el_versions`, helm deps)
- `state.json` — generated by `discover`, then admin-edited
- `RUNBOOK.md` — generated by `build` / `runbook` (gitignored)

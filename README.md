# hauler-airgap-updater

Build and deliver in-place update artifacts for fully air-gapped RKE2 clusters.
See [DESIGN.md](DESIGN.md) for the full design and rationale.

The two sides are completely disconnected. `build` (internet side) produces
**two artifacts** carried across the gap:

- `rpms_<date>.tar.zst` — rke2 RPMs in `el8/`+`el9/` dirs → the **dnf repo
  server** (nodes already point at it).
- `hauler_<date>.tar.zst` — images + helm charts → the **hauler server**.

Python 3, stdlib only. Shells out to `kubectl` / `helm` / `hauler` / `tar`.

## Usage

### 1. discover (internet side, against a reachable reference cluster)
```
./airgap_update.py discover            # writes state.json (review/edit it)
```
Reads RKE2 version (`kubectl get nodes`) and helm-dep versions (`helm list`).
The reference cluster is a *proxy* — confirm the real air-gap cluster's current
version and edit `state.json` before building.

### 2. build (internet side)
```
./airgap_update.py build --dry-run                 # resolve + preview, no downloads
./airgap_update.py build --target 1.34             # interactive deps, real build
./airgap_update.py build --target 1.34 --deps-latest --no-input
```
- `--current TAG`  override the assumed current RKE2 version
- `--target MINOR` e.g. `1.34` (skip the interactive picker)
- `--deps-latest`  take latest for all helm deps
- `--dry-run`      write `airgap_hauler.yaml` + print the RPM list, then stop

### 3. deliver across the gap
- **RPMs:** `tar -xf rpms_<date>.tar.zst -C <dnf-repo-base>` (releasever-split),
  then `createrepo` (or let existing automation do it).
- **Hauler:** on the hauler server, as root:
  ```
  ./airgap_update.py ingest hauler_<date>.tar.zst
  ```

### 4. apply on the nodes (manual, by hand)
Control-plane servers first, **one minor at a time**, wait for `Ready`, then
agents:
```
dnf clean all && dnf update && systemctl restart rke2-server   # agents: rke2-agent
```

## Files
- `config.json` — static config (server paths, `el_versions`, helm deps)
- `state.json` — generated by `discover`, then admin-edited
